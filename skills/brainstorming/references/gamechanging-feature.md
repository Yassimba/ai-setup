# 10x hunt

You were invoked with no idea attached, so surface one worth brainstorming: a move that makes the product 10x more valuable — not 10% better — the kind that makes users say "how did I live without this?" Think like a founder who owns this product.

## Ground yourself

If the user named a product or area, start there; otherwise infer it from the repo and confirm in one question. Then research the codebase and existing features so every idea cites evidence:

- What problem does this solve today, and for whom?
- What's the core action? Where do users spend most time?
- What do users complain about or request most — and is "users want X" actually true, or an assumption?
- Any constraints worth knowing (technical limits, timeline, team size)?

## Generate across three scales

Capture first, evaluate later. Force yourself through each scale:

- **Massive** (high effort, transformative) — expand what the product can do: new markets, new use cases, "platform not tool" moves, things that would make competitors nervous.
- **Medium** (moderate effort, high leverage) — force multipliers on what already works: make the core action 10x faster, exploit unused data, automate painful workflows, turn casual users into power users.
- **Small** (low effort, disproportionate value) — tiny changes that punch above their weight: one button that saves minutes daily, surfacing info users hunt for, one indicator that kills an anxiety.

If stuck, run through these categories:

| Category | Question |
| --- | --- |
| Speed | What takes too long? |
| Automation | What's repetitive? |
| Intelligence | What could be smarter? |
| Integration | What else do users use? |
| Collaboration | How do users work together? |
| Personalization | How is everyone different? |
| Visibility | What's hidden that shouldn't be? |
| Confidence | What creates anxiety? |
| Delight | What would make a user tell a friend? |
| Access | Who can't use this yet? |
| Crazy | What sounds absurd but might work? |

## Evaluate ruthlessly

| Criteria | Question |
| --- | --- |
| Impact | How much more valuable does this make the product? |
| Reach | What % of users would this affect? |
| Frequency | How often would users encounter the value? |
| Differentiation | Does this set us apart or just match competitors? |
| Defensibility | Easy to copy, or does it compound over time? |
| Feasibility | Can we actually build this? |

Score each:

- 🔥 **Must do** — high impact, clearly worth it
- 👍 **Strong** — good impact, should prioritize
- 🤔 **Maybe** — interesting but needs more thought
- ❌ **Pass** — not worth it right now

Ideas must be specific enough to score — "better UX" is not an idea; "one-click rescheduling from the notification" is. Judge by user value, not feature count: ten features at 1% each ≠ one at 10x. If every idea scores "obviously good," you haven't thought hard enough.
